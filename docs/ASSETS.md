# پایپ‌لاین عکس‌های Armory (تسک ۶۱)

عکس‌های واقعی گان‌ها، اتچمنت‌ها و عکس‌های اتمسفریک از **Wikimedia Commons** (لایسنس آزاد) دانلود می‌شوند. فایل‌های باینری در گیت نگهداری نمی‌شوند؛ با یک دستور روی سرور/لوکال گرفته می‌شوند.

## اجرا

```bash
# پیشنهاد: برای خروجی WebP بهینه
pip install Pillow

python scripts/fetch_assets.py
# یا فقط چند آیتم:
python scripts/fetch_assets.py --only m4a1,glock17
```

- خروجی پیش‌فرض: `backend/uploads/armory/` (همان پوشه‌ای که nginx و بک‌اند از مسیر `/uploads` سرو می‌کنند) — مثلاً `/uploads/armory/m4a1.webp`
- منبع، لایسنس و صاحب اثر هر عکس در `assets/manifest.lock.json` ثبت می‌شود
- سیدر (تسک ۶۵) مسیر عکس‌ها را از روی slug می‌سازد: `/uploads/armory/<slug>.webp`

## مدیریت منابع

فایل `assets/manifest.json` لیست آیتم‌هاست:

- `commons_file`: نام دقیق فایل در Commons (اولویت با این است)
- `search`: عبارت جستجوی جایگزین — اگر فایل دقیق تعریف/پیدا نشد، بهترین نتیجه با عرض حداقل ۵۰۰px انتخاب می‌شود
- برای تعویض عکس یک آیتم، فقط `commons_file` را عوض کنید و دوباره اسکریپت را با `--only <slug>` اجرا کنید

## نکات حقوقی (مهم برای پروداکشن)

- بیشتر عکس‌ها Public Domain یا CC هستند. برای موارد **CC BY / CC BY-SA** ذکر منبع الزامی است — یک صفحه «اعتبار تصاویر» در فوتر سایت بسازید و محتوای `manifest.lock.json` را آنجا نمایش دهید (تسک UI جداگانه)
- عکس محصولات سایت سازنده‌ها (Tokyo Marui، G&G و…) بدون اجازه استفاده نشود
- بهترین حالت نهایی: عکاسی اختصاصی از تجهیزات واقعی مجموعه — بعد از تحویل دیتابیس واقعی، فقط فایل‌ها را در همان مسیر جایگزین کنید (نام‌گذاری بر اساس slug ثابت بماند)
