# پلتفرم ایرسافت 🎯

پلتفرم کامل مدیریت مجموعه ایرسافت شامل: رزرو آنلاین سانس، رویدادها و فروش بلیت، لیگ و تیم‌ها و جدول رده‌بندی، فروشگاه، گالری، نظرات کاربران و پنل مدیریت کامل با قابلیت سوییچ درگاه پرداخت (زرین‌پال / زیبال / آیدی‌پی / درگاه تستی).

## تکنولوژی‌ها

| بخش | تکنولوژی |
|---|---|
| بک‌اند | FastAPI (Python 3.12) + SQLAlchemy 2 + PostgreSQL 16 |
| فرانت‌اند | Next.js 14 + TypeScript + Tailwind CSS (کاملاً فارسی و راست‌چین) |
| وب‌سرور | Nginx (ریورس پروکسی) |
| استقرار | Docker Compose |

## ساختار پروژه

```
airsoft-platform/
├── backend/            # API با FastAPI
│   └── app/
│       ├── routers/    # auth, catalog, bookings, events, leagues, shop, payments, admin, uploads
│       ├── payments/   # درگاه‌ها: zarinpal, zibal, idpay, mock
│       ├── models.py   # مدل‌های دیتابیس
│       └── seed.py     # داده اولیه (ادمین + داده نمونه)
├── frontend/           # سایت Next.js
│   └── app/            # صفحات عمومی + پنل ادمین (/admin)
├── nginx/nginx.conf    # ریورس پروکسی
├── docker-compose.yml
└── .env.example        # نمونه تنظیمات محیطی
```

## راه‌اندازی روی سرور (پروداکشن)

### ۱) پیش‌نیازها

یک سرور VPS (ترجیحاً اوبونتو ۲۲.۰۴ به بالا) با Docker و Docker Compose:

```bash
curl -fsSL https://get.docker.com | sh
```

### ۲) انتقال پروژه و تنظیم .env

پروژه را روی سرور کپی کنید (مثلاً در `/opt/airsoft-platform`) و فایل تنظیمات را بسازید:

```bash
cd /opt/airsoft-platform
cp .env.example .env
nano .env
```

مقادیر مهم `.env`:

| کلید | توضیح |
|---|---|
| `POSTGRES_PASSWORD` | یک رمز قوی برای دیتابیس |
| `SECRET_KEY` | کلید امضای توکن‌ها — با دستور `openssl rand -hex 32` بسازید |
| `SITE_URL` | آدرس نهایی سایت مثل `https://example.com` (برای کال‌بک درگاه پرداخت حیاتی است) |
| `ADMIN_PHONE` / `ADMIN_PASSWORD` / `ADMIN_NAME` | حساب مدیر کل که در اولین اجرا ساخته می‌شود |
| `BACKEND_CORS_ORIGINS` | همان دامنه سایت |
| `KAVENEGAR_API_KEY` | اختیاری — برای ارسال پیامک تأیید رزرو (خالی بماند = پیامک غیرفعال) |

### ۳) اجرا

```bash
docker compose up -d --build
```

سرویس‌ها: `db` (پستگرس)، `backend` (پورت داخلی 8000)، `frontend` (پورت داخلی 3000)، `nginx` (پورت 80).

در اولین اجرا جدول‌ها ساخته می‌شوند، حساب ادمین از `.env` ایجاد می‌شود و داده نمونه (دو زمین، سانس‌های یک هفته، تجهیزات و محصولات) seed می‌شود.

بررسی سلامت:

```bash
curl http://localhost/api/health
docker compose logs -f backend
```

### ۴) دامنه و SSL

رکورد A دامنه را به IP سرور بزنید. برای HTTPS ساده‌ترین راه استفاده از certbot در حالت خودکار است:

```bash
sudo apt install certbot python3-certbot-nginx
```

یا اینکه پورت 80 کانتینر nginx را پشت یک nginx میزبان / Cloudflare قرار دهید. بعد از فعال‌سازی HTTPS حتماً `SITE_URL` را در `.env` به `https://` تغییر داده و ری‌استارت کنید:

```bash
docker compose up -d --force-recreate backend frontend
```

### ۵) پیکربندی درگاه پرداخت (از پنل ادمین)

1. با شماره و رمز ادمین وارد شوید: `/login`
2. به **پنل مدیریت ← تنظیمات و درگاه** بروید.
3. درگاه فعال را انتخاب کنید و کلیدهای آن را وارد و ذخیره کنید:
   - **زرین‌پال**: Merchant ID (۳۶ کاراکتری). برای تست، سندباکس را فعال بگذارید.
   - **زیبال**: مقدار Merchant. برای تست کافی است `zibal` وارد کنید.
   - **آیدی‌پی**: API Key پنل آیدی‌پی + گزینه سندباکس.
   - **درگاه تستی (Mock)**: بدون نیاز به کلید؛ صفحه پرداخت شبیه‌سازی‌شده با دکمه موفق/ناموفق — مناسب تست کامل چرخه خرید قبل از گرفتن درگاه واقعی.
4. درگاه هر لحظه از همین صفحه قابل تعویض است؛ تنظیمات در دیتابیس ذخیره می‌شود و نیازی به ری‌استارت نیست.

آدرس کال‌بک که باید در پنل درگاه (در صورت نیاز) ثبت کنید: `SITE_URL/api/payments/callback/...` — به صورت خودکار در هر تراکنش ارسال می‌شود و معمولاً نیازی به ثبت دستی نیست.

### ۶) نقش‌ها و پنل مدیریت

| نقش | دسترسی |
|---|---|
| `admin` (مدیر کل) | همه‌چیز + تنظیمات درگاه + تغییر نقش کاربران |
| `staff` (کارکنان) | مدیریت رزروها، سانس‌ها، رویدادها، سفارش‌ها، نظرات، گالری |
| `player` (بازیکن) | کاربر عادی سایت |

بخش‌های پنل: داشبورد (آمار و درآمد)، رزروها (تغییر وضعیت/حضور)، سانس‌ها، زمین‌ها (با آپلود تصویر)، تجهیزات اجاره‌ای، رویدادها (+ لیست بلیت‌های هر رویداد)، لیگ و مسابقات (ثبت نتیجه = به‌روزرسانی خودکار جدول)، محصولات، سفارش‌ها (پرداخت‌شده ← ارسال ← تحویل)، کاربران، نظرات (تأیید/حذف)، گالری، تراکنش‌ها، تنظیمات.

### ۷) پشتیبان‌گیری

```bash
# بکاپ دیتابیس
docker compose exec db pg_dump -U airsoft airsoft > backup-$(date +%F).sql

# بکاپ فایل‌های آپلودشده
docker cp airsoft-backend:/app/uploads ./uploads-backup
```

بازیابی: `cat backup.sql | docker compose exec -T db psql -U airsoft airsoft`

پیشنهاد: این دو دستور را در cron روزانه بگذارید.

### ۸) به‌روزرسانی

```bash
cd /opt/airsoft-platform
docker compose up -d --build
```

## نکات مهم

- **واحد پول**: همه قیمت‌ها به تومان وارد و نمایش داده می‌شوند (تبدیل به ریال برای درگاه‌ها به‌صورت خودکار انجام می‌شود).
- **زمان سانس‌ها**: زمان‌ها در دیتابیس به UTC ذخیره می‌شوند و در سایت به وقت محلی کاربر (تهران) نمایش داده می‌شوند. هنگام ثبت سانس در پنل ادمین، زمان را به UTC وارد کنید (تهران منهای ۳:۳۰).
- **فرم ایمنی (Waiver)**: هنگام رزرو، امضای دیجیتال فرم رضایت‌نامه اجباری است و متن آن از «تنظیمات ← قوانین» خوانده می‌شود.
- **داده نمونه**: پس از راه‌اندازی واقعی، زمین‌ها/سانس‌ها/محصولات نمونه را از پنل ادمین ویرایش یا غیرفعال کنید.
- این پروژه در محیط بدون اینترنت ساخته شده؛ کد بک‌اند با `py_compile` بررسی شده اما اولین `docker compose up --build` روی سرور شما بیلد کامل (نصب پکیج‌ها و بیلد Next.js) را انجام می‌دهد. اگر خطایی در بیلد دیدید، لاگ همان مرحله را بررسی کنید.

## عیب‌یابی سریع

| مشکل | راه‌حل |
|---|---|
| صفحه ۵۰۲ | `docker compose logs backend frontend` — احتمالاً سرویس هنوز بالا نیامده |
| پرداخت به سایت برنمی‌گردد | `SITE_URL` در `.env` را بررسی کنید و سرویس‌ها را recreate کنید |
| پیامک ارسال نمی‌شود | `KAVENEGAR_API_KEY` را تنظیم کنید (اختیاری است) |
| رمز ادمین فراموش شد | مقدار `ADMIN_PASSWORD` در `.env` را عوض کرده و backend را ری‌استارت کنید — رمز ادمین seed به‌روز می‌شود |
